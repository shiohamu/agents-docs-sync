# フェーズ6: ドキュメントのレビュー手順書

## レビューの目的
プロジェクトのドキュメントの完全性、正確性、最新性を評価し、ユーザーと開発者にとって有用なドキュメントであることを確認します。

## 対象ファイル・ディレクトリ

### メインドキュメント
- `README.md` - プロジェクトのメインドキュメント
- `AGENTS.md` - AIエージェント向けドキュメント
- `REVIEW_REPORT.md` - レビュー報告書（既存）

### 実装ドキュメント
- `docs/api.md` - APIドキュメント
- `docs/implementation/` - 実装ガイド
  - `IMPLEMENTATION_GUIDE.md` - 実装ガイド
  - その他の実装ドキュメント

### コード内ドキュメント
- モジュールのdocstring
- クラスのdocstring
- 関数のdocstring

## レビュー手順

### ステップ1: README.mdのレビュー

#### 1.1 プロジェクト概要
- [ ] プロジェクトの説明が明確で正確か
- [ ] 主な機能が適切に記載されているか
- [ ] 使用技術が正確に記載されているか
- [ ] プロジェクトの目的が明確か

#### 1.2 セットアップ手順
- [ ] 必要な環境が明確に記載されているか（Pythonバージョン、依存関係など）
- [ ] インストール手順が正確で実行可能か
- [ ] 複数のインストール方法が提供されているか（uv, pip, pyproject.toml）
- [ ] 初回設定の手順が明確か（`config.yaml`の作成など）

#### 1.3 使用方法
- [ ] 基本的な使用方法が記載されているか
- [ ] コマンドライン引数の説明が適切か
- [ ] 使用例が提供されているか

#### 1.4 プロジェクト構造
- [ ] プロジェクト構造の説明が正確か
- [ ] ディレクトリの説明が適切か
- [ ] 主要なファイルの説明が適切か

#### 1.5 ドキュメントの整合性
- [ ] 記載されているファイルパスが実際に存在するか
- [ ] 記載されているコマンドが実際に動作するか
- [ ] 自動生成セクションと手動セクションが適切に区別されているか（`<!-- MANUAL_START:... -->`、`<!-- MANUAL_END:... -->`）

#### 1.6 その他の情報
- [ ] ライセンス情報が記載されているか（必要に応じて）
- [ ] コントリビューションガイドが記載されているか（必要に応じて）
- [ ] トラブルシューティング情報が記載されているか（必要に応じて）

### ステップ5: コード内ドキュメントのレビュー

#### 5.1 モジュールのdocstring
- [ ] 各モジュールにdocstringが記載されているか
- [ ] モジュールの目的が明確に説明されているか
- [ ] 主要なクラスと関数がリストされているか

#### 5.2 クラスのdocstring
- [ ] 各クラスにdocstringが記載されているか
- [ ] クラスの目的が明確に説明されているか
- [ ] 主要なメソッドがリストされているか
- [ ] 使用例が提供されているか（必要に応じて）

#### 5.3 関数のdocstring
- [ ] 各関数にdocstringが記載されているか
- [ ] 関数の目的が明確に説明されているか
- [ ] パラメータの説明が適切か（型、説明、デフォルト値）
- [ ] 戻り値の説明が適切か（型、説明）
- [ ] 例外の説明が適切か（発生する可能性のある例外）
- [ ] 使用例が提供されているか（必要に応じて）

#### 5.4 docstringの形式
- [ ] docstringの形式が一貫しているか（Google形式、NumPy形式など）
- [ ] プロジェクト全体で同じ形式が使用されているか

### ステップ6: ドキュメントの最新性の確認

#### 6.1 コードとの整合性
- [ ] ドキュメントが最新のコードと一致しているか
- [ ] 削除された機能のドキュメントが残っていないか
- [ ] 追加された機能のドキュメントが記載されているか

#### 6.2 更新日時の確認
- [ ] ドキュメントの更新日時が記載されているか
- [ ] 更新日時が適切か（最近更新されているか）

### ステップ7: ドキュメントの品質確認

#### 7.1 可読性
- [ ] ドキュメントが読みやすいか（適切な見出し、段落、リスト）
- [ ] 専門用語が適切に説明されているか
- [ ] コード例が適切にフォーマットされているか

#### 7.2 完全性
- [ ] 必要な情報がすべて記載されているか
- [ ] 不足している情報がないか

#### 7.3 正確性
- [ ] 記載されている情報が正確か
- [ ] タイポや誤字がないか
- [ ] リンクが正しく機能するか（内部リンク、外部リンク）

## 確認チェックリスト

### 高優先度
- [ ] README.mdの内容が正確で最新
- [ ] AGENTS.mdがOpenAI仕様に準拠している
- [ ] セットアップ手順が正確で実行可能
- [ ] コード内ドキュメント（docstring）が充実している

### 中優先度
- [ ] APIドキュメントが完全で正確
- [ ] 実装ドキュメントが適切に整理されている
- [ ] 使用例が提供されている

### 低優先度
- [ ] ドキュメントの詳細度
- [ ] トラブルシューティング情報
- [ ] コントリビューションガイド
