## レビューの実施方法

1. **順序通りに実施**: フェーズ1から順番に実施することを推奨します
2. **チェックリストの使用**: 各手順書のチェックリストを使用して、漏れなく確認します
3. **結果の記録**: 発見された問題と強みを記録します
4. **レビュー報告書の更新**: すべてのフェーズ完了後、`REVIEW_REPORT.md`を更新します

## レビュー結果の記録

各フェーズのレビュー結果は、以下の形式で記録します：

### 問題の記録
```markdown
#### 問題番号: [問題のタイトル]
- **重要度**: 高/中/低
- **場所**: [ファイルパス:行番号]
- **説明**: [問題の詳細説明]
- **影響**: [問題が及ぼす影響]
- **推奨対応**: [推奨される修正方法]
```

### 強みの記録
```markdown
#### 強み: [強みのタイトル]
- **場所**: [該当箇所]
- **説明**: [なぜ良いのか]
```

## レビュー完了後の作業

すべてのフェーズのレビュー完了後：

1. **レビュー報告書の更新**: `REVIEW_REPORT.md`に最新のレビュー結果を反映
2. **優先度別の改善提案**: 高/中/低の優先度で改善提案を整理
3. **次のステップの決定**: 改善項目の実装順序を決定

## 注意事項

- 各フェーズは独立して実施可能ですが、順序通りに実施することを推奨します
- レビュー中に発見された問題は、即座に修正せず、まず記録します
- レビュー完了後、優先度に基づいて修正を実施します
