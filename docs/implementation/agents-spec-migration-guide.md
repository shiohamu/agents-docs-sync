# AGENTS.md仕様変更実装手順書

この手順書では、OpenAI主導のAGENTS.md仕様に準拠し、API/ローカルLLMの両パターンを網羅した指示書を生成するように変更する手順を説明します。

## 前提条件

- Python 3.12以上がインストールされていること
- 既存のドキュメント生成システム（`.docgen`）が動作していること
- プロジェクトのビルド/テスト手順が明確に定義されていること

## 実装ステップ一覧

各ステップの詳細な手順は、以下の個別ファイルを参照してください：

1. **[ステップ9: プロジェクト情報収集モジュールの作成](./step9-project-info-collector.md)**
   - ビルド/テスト手順、コーディング規約などの情報を収集するモジュールを作成

2. **[ステップ10: AGENTS.md生成設定の拡張](./step10-agents-config-extension.md)**
   - AGENTS.md生成用の設定セクションを追加

3. **[ステップ11: AGENTS.mdテンプレートシステムの実装](./step11-agents-template.md)**
   - OpenAI仕様準拠のAGENTS.mdテンプレートを作成

4. **[ステップ12: AgentsGeneratorの変更](./step12-agents-generator-refactor.md)**
   - 新しい仕様に合わせてAgentsGeneratorを大幅に変更

5. **[ステップ13: AGENTS.md生成のテストと検証](./step13-agents-testing.md)**
   - 各機能の動作確認とテストの作成

## トラブルシューティング

詳細は各ステップのドキュメントを参照してください。一般的な問題については以下を確認：

- **AGENTS.mdが生成されない**: ステップ9、10、12の確認事項を再確認
- **プロジェクト情報が正しく収集されない**: ステップ9の確認事項を再確認
- **テンプレートが正しく表示されない**: ステップ11、12の確認事項を再確認
- **API/ローカルLLMのパターンが表示されない**: ステップ10の設定を再確認

## まとめ

この手順書に従って実装することで、以下の機能が実現されます：

- ✅ OpenAI仕様に準拠したAGENTS.mdの生成
- ✅ API/ローカルLLMの両パターンのサポート
- ✅ プロジェクト情報の自動収集
- ✅ コーディング規約の自動検出
- ✅ カスタマイズ可能な設定

各ステップを順番に実行し、各段階で動作確認を行うことで、問題を早期に発見し、スムーズに実装を完了できます。
